在尝试修复这 2 个问题时，得到社区的反馈：留着它们吧。

能不用 LRU hash map 就别用

问题 1：丢数据

早有耳闻：LRU hash map 会丢数据。

在给 percpu map CPU flags 编写自测用例时，就遇上了丢数据问题：当 LRU hash map 满了时，再次 update_elem() 就会丢数据，即使想要更新的是已有条目。
